# Release Channels and Promotion Policy

Status: channel framework; no stable channel authorized  
Owner: release, security, quality, support, and program

Proposed channels are `nightly`, `research`, `developer-preview`, `beta`, and `stable`. Each has distinct audience, update cadence, evidence, data compatibility, support, rollback, and retention obligations.

Promotion is artifact-based. The exact tested artifact advances; it is not rebuilt after approval without repeating provenance and release checks.

Stable promotion requires all blocking release gates, error-budget compliance, signed metadata, rollback readiness, migration evidence, incident staffing, release notes, known-issue disclosure, and human release authorization.
