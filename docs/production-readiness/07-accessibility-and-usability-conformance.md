# Accessibility and Usability Conformance

Status: release-gate framework  
Owner: accessibility, product, platform, quality, and support

Automated tools assist but do not establish accessibility conformance. Stable release requires knowledgeable human evaluation and supported assistive-technology workflows.

The matrix covers keyboard, screen readers, voice control, switch control, magnification, high contrast, forced colors, reduced motion, text scaling, IME, RTL, security prompts, agent confirmation, update/recovery, DevTools, Spaces, split views, and resource controls.

Critical defects in origin, permission, credential, confirmation, navigation, update, or recovery workflows block release.
