# Human Handoff and Escalation

Status: operating baseline  
Owner: program, security, architecture, legal, release, and support

Escalate when a task encounters:

- ambiguous or conflicting requirements;
- a new trust boundary;
- a dependency or license decision;
- a security finding or possible exploit;
- user-data loss or migration risk;
- performance/security tradeoff outside accepted limits;
- platform or standards ambiguity;
- requested authority outside the task manifest;
- repeated non-reproducible failure;
- release, signing, disclosure, legal, or support decisions;
- insufficient independent review capacity.

A handoff contains current commit, task state, evidence, failed approaches, unresolved questions, risks, proposed next action, and exact authority required. The agent does not guess the decision.
